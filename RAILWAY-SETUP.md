# Railway Setup Guide

## Prerequisites

- Railway account with billing configured
- Railway CLI installed: `npm i -g @railway/cli`

## 1. Create Project

```bash
railway init falsisters
railway environment production
```

## 2. PostgreSQL 16

In Railway dashboard, add a PostgreSQL service:

1. Click **+ New** → **Database** → **PostgreSQL**
2. Railway auto-generates connection strings as environment variables
3. No manual config needed — the `DATABASE_URL` will be available to all services
4. To get the URL for external tools (pg_dump, etc.):
   ```bash
   railway variables --service postgres
   ```

## 3. MinIO (S3-compatible Storage)

1. Click **+ New** → **Empty Service**
2. Configure:
   - **Name:** `minio`
   - **Image:** `minio/minio:latest`
   - **Command:** `server /data --console-address :9001`
3. Set environment variables:
   ```
   MINIO_ROOT_USER = falsisters
   MINIO_ROOT_PASSWORD = <generate-strong-password-here>
   MINIO_BROWSER = on
   ```
4. Deploy the MinIO service
5. Open the MinIO console (port 9001, use Railway's public URL or `railway open`)
6. In the MinIO Console:
   - Create a bucket named `falsisters-bucket`
   - Set bucket policy to **private**
   - Go to **Access Keys** → **Create Access Key**
   - Save the generated Access Key and Secret Key

## 4. Server

1. Click **+ New** → **GitHub Repo**
2. Select the `falsisters-server` repository
3. Set **Root Directory** to `server`
4. Set **Builder** to `Dockerfile` (already in repo)
5. Set environment variables in Railway dashboard:

   ```
   # Auto-generated by Railway (Postgres service reference):
   DATABASE_URL = ${{Postgres.DATABASE_URL}}

   # MinIO S3 (from step 3):
   S3_ENDPOINT = https://<minio-service-name>.railway.app
   S3_REGION = us-east-1
   S3_ACCESS_KEY_ID = <minio-access-key>
   S3_SECRET_ACCESS_KEY = <minio-secret-key>
   S3_BUCKET_NAME = falsisters-bucket
   S3_FORCE_PATH_STYLE = true

   # JWT:
   JWT_SECRET = <generate-strong-random-string-at-least-32-chars>

   # Server port:
   PORT = 3001
   ```

   > **Important:** Leave `SUPABASE_*` env vars **empty** on Railway. When absent, the server auto-switches to MinIO signed URLs instead of Supabase public URLs.

## 5. Environment Variables Reference

| Variable | Source | Notes |
|----------|--------|-------|
| `DATABASE_URL` | Railway Postgres auto | Connection string with pool |
| `S3_ENDPOINT` | MinIO Railway URL | e.g., `https://minio.railway.app` |
| `S3_REGION` | Manual | `us-east-1` |
| `S3_ACCESS_KEY_ID` | MinIO Console | Generated access key |
| `S3_SECRET_ACCESS_KEY` | MinIO Console | Generated secret key |
| `S3_BUCKET_NAME` | Manual | `falsisters-bucket` |
| `S3_FORCE_PATH_STYLE` | Manual | `true` for MinIO |
| `JWT_SECRET` | Manual | Random 32+ char string |
| `PORT` | Manual | `3001` |

## 6. Deploy

```bash
railway up
```

After first deploy, run database migrations:

```bash
railway run --service server yarn prisma migrate deploy
```

## 7. Verify

Check the server health endpoint:
```
GET https://<server-service>.railway.app/health
```

Should return: `{ "status": "ok" }`

## 8. Next Steps

After Railway is running:
1. Run `scripts/migrate-db.sh` to migrate data from Supabase → Railway PG
2. Run `scripts/migrate-storage.ts` to migrate files from Supabase → MinIO
3. Point backoffice `API_URL` to the new Railway server URL
4. Update mobile app API base URL
5. After 1 week of stable operation, delete the Supabase project
